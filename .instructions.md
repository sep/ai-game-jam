# NCWIT Indiana AIC Awards Event — Game Arcade

This is a repo for a STEM event where high school students create Three.js games using GitHub Copilot.

## Fast Path For build-my-game
- When the user invokes `/build-my-game` or asks to create a game, start by asking the participant interview questions immediately.
- Ask the first question exactly as: `What is your name?`
- Ask for the game title last, after the gameplay/theme questions.
- Use a friendly and supportive tone during the interview.
- Do not inspect unrelated files before the interview.
- After answers are collected, only read the minimum required files: `games/_template/index.html` and `games/manifest.js`.
- Do not read `README.md`, `index.html`, or `styles.css` unless the user asks to change the arcade landing page or facilitator docs.
- Treat this as a narrow single-game generation flow, not a repo-wide analysis task.

## Key Rules
- All games are single-file HTML (`games/<name>/index.html`) with inline JS/CSS
- Game-specific styling must stay inside each participant's own `games/<name>/index.html`
- Do not create per-game external CSS files
- Do not modify the root `index.html` or `styles.css` when building or customizing a participant game unless the user explicitly asks to change the arcade site itself
- Three.js is loaded from CDN: `https://cdnjs.cloudflare.com/ajax/libs/three.js/r128/three.min.js`
- No npm, no build tools, no external dependencies beyond CDN-hosted Three.js
- Games must work when opened directly in a browser (no server required)
- When creating a new game, always update `games/manifest.js` with the new entry

## Repo Structure
- `games/_template/index.html` — reference game template
- `games/manifest.js` — registry of all participant games (JS file: `const GAMES = [...]`)
- `games/<participant>/index.html` — individual participant games
- `index.html` — landing page that loads manifest.js via script tag to show all games
